**Story:** Show HN: BetterClaw - Compile a paragraph into a workflow that gates agent tools (by infamous-oven, 2 points, 0 comments at draft time)

**OP:** OP describes building BetterClaw after the April 25 PocketOS incident (Cursor agent running Claude deleted a company's prod DB in 9 seconds via a Railway MCP server, then zapped the backups; "be careful" in the system prompt didn't bind). BetterClaw's pitch: write the workflow in plain English ("diagnose the credential mismatch - read the config, test the connection, do not modify or delete anything"), the CLI compiles the paragraph into a directed graph where each node declares allowed tools, and a hook blocks anything outside the graph before it dispatches to the MCP. Repo includes a runnable PocketOS reproducer with a mock Railway server. OP explicitly asks for feedback on three things: (1) is paragraph -> graph the right authoring model vs YAML / a DSL, (2) where does this fall down (multi-step approvals, loops, sub-agents), (3) what other agent runtimes should they support beyond Claude Code + Cowork + OpenClaw.

**My reply:**

```
(disclosure: I work on FailProof AI: https://github.com/exospherehost/failproofai)

The PocketOS reproducer in the repo is a strong move - turns "this is theoretical" into "I watched it run on my laptop."

On paragraph -> graph: reads well for diagnose-only flows where the steps enumerate naturally. Where I'd expect it to bite is when the agent legitimately deviates - retrying a different read tool, exploring before fixing. Once the graph compiles, the agent is on rails; if the paragraph missed a branch, you're recompiling.

For "where it falls down": sub-agents look like the hardest case either way. The child's tool budget has to be carved out of the parent somehow.

We went the other way: per-intent policy functions returning allow / deny / instruct on every tool call, no prescribed workflow shape. Trade-off is no single-paragraph contract for what a run is allowed to do. Did you consider that authoring model and reject it, or do you prefer the graph framing on its own merits?
```
